# Glot500: Scaling Multilingual Corpora And Language Models To 500 Languages

Source: `docs/survey/2305.12182v2.pdf`

## 핵심 내용

- Glot500은 XLM-R 계열을 수평적으로 확장해 511개 언어를 다루려는 프로젝트다.
- 모델 측면에서는 `xlm-roberta-base` 구조를 유지하면서 continued pretraining을 수행한다.
- corpus 측면에서는 Glot500-c를 구축하고, 저자원 언어를 포함하도록 data cleaning과 language/script 관리를 수행한다.
- Glot500-m은 XLM-R baseline보다 여러 task에서 개선을 보였지만, 성능은 corpus size, script, 관련 언어의 도움, 모델 capacity가 함께 결정한다.

## 방법론 포인트

- Vocabulary extension은 tokenizer를 완전히 교체하는 것이 아니라 기존 XLM-R vocabulary를 보존하면서 새 token을 추가하는 방향이다.
- Continued pretraining objective는 MLM이다.
- 언어 sampling은 corpus size에 temperature를 적용하는 방식으로 해석할 수 있다.
- 언어 단위보다 language-script 단위 관리가 중요하다. 같은 언어라도 script가 다르면 tokenization 병목이 다르게 나타날 수 있다.

## Second_try에 가져올 것

- Base model을 `xlm-roberta-base`로 두고, 기존 token id를 보존한 append-style vocab extension을 한다.
- target language corpus를 정리한 뒤 tokenizer training, merge, MLM adaptation 순서로 진행한다.
- tokenization metric만 보지 않고 downstream representation task까지 본다.
- Coptic/Syriac/Cherokee/Ojibwa처럼 script bottleneck이 예상되는 언어와 Latin-script 저자원 언어를 함께 둔다.

## 주의점

- Glot500 규모의 vocab 확장과 학습 step을 그대로 따라가면 second_try compute에 비해 과하다.
- Glot500 본실험에서 random init이 가능했던 것은 대규모 데이터와 긴 학습 때문일 수 있다.
- Bible 기반 결과만으로 일반 domain 성능을 주장하면 과장될 수 있다.

## Plan 반영

- XLM-R tokenizer id 보존.
- `8k`, `16k`, `32k` 작은 vocab grid.
- MLM adaptation 후 encoder-only downstream task 평가.
- final report에는 corpus/domain limitation을 명시.
