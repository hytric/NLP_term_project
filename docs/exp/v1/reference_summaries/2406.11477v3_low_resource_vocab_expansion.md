# How Can We Effectively Expand The Vocabulary Of LLMs With 0.01GB Of Target Language Text?

Source: `docs/survey/2406.11477v3.pdf`

## 핵심 내용

- 저자원 target text 약 `30K sentences` 또는 `0.01GB`만으로 vocabulary expansion을 수행하는 전략을 다룬다.
- 비영어/저자원 언어에서는 기존 tokenizer가 over-fragmentation을 일으켜 inference step과 cost가 증가한다.
- Vocabulary expansion은 inference 효율과 downstream 성능 사이의 tradeoff를 갖는다.
- 저자원 설정에서는 단순 random initialization과 무작정 full continued pretraining이 안정적이지 않다.

## 방법론 포인트

- 새 token embedding은 random보다 Mean 또는 Align 계열 초기화가 더 안정적이다.
- 작은 target corpus에서는 새 token 수가 너무 크면 새 embedding이 충분히 학습되지 않는다.
- Continued pretraining은 full fine-tuning뿐 아니라 layer-selective update, LoRA, mixed strategy를 고려할 수 있다.
- 평가는 downstream score만 아니라 tokenization fragmentation, perplexity, speed/sequence length, 기존 능력 유지까지 같이 봐야 한다.

## Second_try에 가져올 것

- Random은 하한선 baseline으로만 둔다.
- Mean/FVT/Align/Focus 같은 구조적 초기화를 우선 비교한다.
- vocab size grid를 돌리되, added token 수와 downstream 성능을 함께 비교한다.
- zero-step MLM loss와 fixed-step MLM loss를 모두 기록한다.

## 주의점

- 논문은 generation/inference speed까지 다루지만 second_try는 encoder-only downstream으로 재해석해야 한다.
- 너무 큰 vocab은 tokenization은 좋아져도 downstream에서 under-trained token 문제를 만들 수 있다.
- target-only adaptation은 기존 XLM-R multilingual 능력을 잊게 할 수 있다.

## Plan 반영

- `8k`, `16k`, `32k` 모두 비교.
- init comparison은 3 seeds.
- 최종 best checkpoint와 original XLM-R는 downstream 3 seeds.
- report에 forgetting risk와 proxy task 한계를 명시.
