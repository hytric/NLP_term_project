# XLM-R 기반 저자원 언어 Vocabulary Extension에서의 Embedding Initialization 비교

**Term Project Report (한글본)**

---

## 초록 (Abstract)

Multilingual masked language model인 XLM-R은 100여 개 언어를 커버하지만, 학습에 포함되지 않은 tail language에서는 subword가 과도하게 잘게 쪼개지고(over-fragmentation) 표현이 정렬되지 않는 문제가 남는다. Glot500은 mixed corpus로 SentencePiece unigram tokenizer를 다시 학습해 vocabulary를 확장하고, 확장된 모델을 MLM으로 continued pretraining하여 이 문제를 완화했다. 그러나 vocabulary를 확장하면 embedding matrix 뒤에 기존에 없던 새 token row가 추가되는데, 이 row를 **어떤 벡터로 시작할지**는 별도의 modeling 선택으로 남아 있고 체계적으로 비교된 적이 드물다.

본 보고서는 이 선택 하나만 통제 변인으로 두고 비교한다. Base model은 `xlm-roberta-base`, 데이터는 92개 XLM-R-seen replay 언어와 7개 XLM-R-unseen target 언어(Target7)이며, tokenizer는 Glot500 방식의 SentencePiece unigram append로 **고정**한다. 그 위에서 새 token embedding row 초기화만 `random`, `mean`, `FVT`, `weighted FVT`, `family-aware mean`의 다섯 가지로 바꾸어, 동일한 corpus·objective·schedule·evaluation protocol 아래 50K-step까지 continued MLM pretraining을 수행한다.

주요 관찰은 세 가지다. 첫째, 확장 tokenizer는 Target7의 tokens/word를 2.204에서 1.592로 약 27.75% 낮춰 fragmentation을 실제로 줄인다. 둘째, 같은 tokenizer를 공유함에도 초기화에 따라 50K 수렴 loss가 갈린다. 최종 MLM loss는 `weighted FVT`(2.73)·`FVT`(2.76)가 가장 낮고 `random`(3.12)·`mean`(3.27)이 가장 높으며, 특히 `mean`은 `random`보다도 나쁘다. 셋째, 50K downstream에서 우리가 추가한 두 refinement(`weighted FVT`, `family-aware mean`)가 target pseudoperplexity·Tatoeba retrieval·NER·roundtrip alignment에서 최고이며 — 특히 `family-aware mean`은 NER에서 full Glot500-m을 능가한다 — 학습 초반엔 plain `FVT`가 앞서다가 수렴 구간에서 이 refinement에 추월당한다.

claim은 controlled setup 안으로 제한한다. 특정 초기화가 모든 task에서 항상 최고라고 주장하지 않으며, Target7이 모두 Latin script이므로 script 다양성 결론도 내리지 않는다. 본 보고서의 기여는 새 아키텍처가 아니라, low-resource vocabulary adaptation에서 흔히 임의로 정해지는 embedding initialization이 통제된 조건에서도 수렴과 평가 지표를 바꾸며, FVT를 정교화한 초기화가 종합적으로 가장 유리함을 재현 가능하게 문서화한 데 있다.

## 1. 서론 (Introduction)

### 1.1 Vertical scaling vs. horizontal scaling

대규모 언어모델의 발전은 크게 두 방향으로 나뉜다. **Vertical scaling**은 파라미터 수·학습 데이터 양·연산량을 키워 같은 대상을 더 잘 푸는 방향이다(scaling law 계열). **Horizontal scaling**은 더 많은 언어·도메인·태스크를 포괄하는 방향, 즉 "더 많은 대상을 지원할 수 있는가"를 묻는다. Low-resource language 지원은 전형적인 horizontal scaling 문제다.

XLM-R은 100개 언어를 하나의 MLM으로 학습한 강력한 multilingual encoder지만, 학습에 포함되지 않은 언어(XLM-R-unseen tail language)에서는 두 가지 약점이 남는다. (1) **Token fragmentation**: tokenizer가 해당 언어를 본 적이 없어 한 단어가 여러 subword로 과하게 쪼개진다. (2) **Representation mismatch**: 그 언어의 어휘·형태 정보가 embedding 공간에 정렬돼 있지 않다.

### 1.2 선행 해법: Glot500

Glot500은 horizontal scaling을 위해 두 단계를 결합한다. 먼저 500여 language-script의 mixed corpus로 SentencePiece unigram tokenizer를 다시 학습하여 XLM-R의 vocabulary를 확장하고(**vocabulary extension**), 확장된 모델을 그 corpus 위에서 MLM으로 이어 학습한다(**continued pretraining**). Glot500은 결과를 high-resource(head)와 low-resource(tail)로 나눠 보고하여 두 그룹의 이득이 다르다는 점을 드러냈다. 본 보고서도 이 head/tail 보고 방식을 그대로 가져온다.

### 1.3 빈틈: embedding initialization은 별도의 선택이다

Vocabulary를 확장하면 embedding matrix 뒤에 기존에 없던 새 token row가 추가된다. 이 새 row를 어떤 벡터로 시작할지는 tokenizer 설계와 독립적인 **modeling 선택**이다. 기존 방식은 보통 이를 단순 random으로 처리하지만, `random`(무정보), `mean`(기존 embedding 중심), `FVT`(새 token을 기존 tokenizer로 다시 쪼개 그 source subtoken embedding을 재사용) 등 여러 대안이 있다.

본 보고서는 여기에 두 가지 초기화를 **추가로 제안·비교**한다. `weighted FVT`는 FVT의 subtoken 평균을 균등 평균 대신 **표면 길이로 가중**한다(긴 source subtoken이 더 구체적 lexical 신호를 담는다는 가정). `family-aware mean`은 표면형 분해 대신, 새 token의 corpus provenance가 가리키는 **language family의 source-token 평균**을 쓴다(철자보다 계통 prior가 저자원 token에 더 좋은 출발점을 주는지 확인). 이로써 비교 축은 (무정보) `random` → (전역 centroid) `mean` → (표면형 재사용) `FVT` → (표면형+길이 가중) `weighted FVT` → (계통 prior) `family-aware mean`의 다섯 방법으로 확장된다.

### 1.4 실험 설정

`xlm-roberta-base`를 base로, 92개 XLM-R-seen replay 언어와 7개 XLM-R-unseen target(Target7)을 쓴다. tokenizer는 Glot500 방식으로 고정하고, **새 embedding row 초기화만** 다섯 방법으로 바꿔 continued MLM pretraining을 돌린다. 따라서 결과의 method 간 차이는 tokenizer가 아니라 initialization과 그에 따른 training dynamics에서 온다.

### 1.5 연구 질문과 기여

- **RQ1.** Glot500-style tokenizer extension이 Target7의 token fragmentation을 실제로 줄이는가?
- **RQ2.** 같은 tokenizer·corpus에서 embedding initialization이 50K-step MLM 수렴에 영향을 주는가?
- **RQ3.** 초기화 차이가 downstream 지표에서 나타나는가, 어느 task family에서 두드러지는가?
- **RQ4.** 학습 초반(10K) 순위와 수렴(50K) 순위가 일치하는가, 뒤바뀌는가?

**기여.** (1) tokenizer를 고정하고 embedding initialization만 통제 비교하는 재현 가능한 ablation 설계, (2) `weighted FVT`·`family-aware mean` 두 초기화 방법 제안, (3) 초기화가 50K 수렴 loss를 유의하게 바꾸며 그 이득이 수렴까지 유지됨을 입증, (4) 수렴 시점 downstream에서 두 refinement가 plain FVT를 추월하고 `family-aware mean`이 NER에서 Glot500-m까지 능가함을 발견, (5) 문장 표현 공간의 계통 구조 진단, (6) downstream task의 정당화와 coverage 투명성. 단 "Glot500 완전 재현", "모든 low-resource 일반화", "script diversity 대표"는 주장하지 않는다.

## 2. 관련 연구 (Related Work)

**XLM-R.** 본 실험의 base encoder이자 source tokenizer(250,002 vocab)다. 아키텍처를 새로 제안하지 않으며, 기존 token은 **token string identity 기준으로 source row를 보존**하고 확장으로 추가되는 새 row만 실험 대상으로 삼는다. SPM에 piece를 append하면 `<mask>` 같은 special token id가 이동할 수 있어(250001→366665) id가 아니라 string으로 remap해야 한다.

**Glot500.** 여기서 (1) mixed corpus의 SentencePiece unigram tokenizer 재학습, (2) append-only vocab injection, (3) continued MLM pretraining, (4) head/tail 보고를 가져온다. 본 보고서는 500개 전체 확장이 아니라 **92 seen + 7 target** 축소 설정이며, 질문은 scaling이 아니라 **initialization**이다.

**Vocabulary transfer / initialization.** `FVT`(Fast Vocabulary Transfer)는 새 token 표면형을 source tokenizer로 재분해해 구성 subtoken embedding 평균을 새 row로 쓴다(main hypothesis). WECHSEL·FOCUS 계열은 lexical overlap·보조 임베딩 기반 transfer의 근거이며, Hewitt류의 embedding 안정성 논의는 `mean` baseline의 배경이다. `weighted FVT`·`family-aware mean`은 본 실험의 local variant다.

**SentencePiece 설정.** `model_type=unigram`(XLM-R/Glot500 계열과 일치), `byte_fallback=true`(드문 문자를 byte로 안전 분해), `character_coverage=1.0`(Target7 문자 보존).

**평가 프로토콜.** Glot500 Table 3 downstream suite + pseudoperplexity를 따른다(본 보고서는 sentence retrieval·NER·roundtrip·text classification과 PPPL을 보고). Glot500은 저자원 언어에 labeled train set이 없음을 명시하고, supervised task를 **English로 fine-tune 후 target zero-shot 전이**하는 XTREME 방식으로 평가한다(원문: *"Since training data does not exist for some languages, we finetune on English ... and evaluate zero-shot transfer."*). 본 보고서도 동일 프로토콜·데이터를 써서 XLM-R·Glot500-m baseline과 같은 축에서 비교된다. **왜 English zero-shot인가**: tail 언어에 labeled 데이터가 없어 target fine-tune이 불가능하고, English로만 supervision을 주면 target 성능이 나오는 유일한 경로가 **공유 multilingual 표현의 cross-lingual 정렬**뿐이라, 이 설정이 곧 vocabulary extension·initialization이 바꾸는 표현 정렬을 직접 측정한다.

### 3.2 왜 이 언어들인가

언어 선택은 **"평가까지 연결 가능한 tail language"** 기준이다. 대부분의 진짜 저자원 언어는 human-labeled downstream 데이터가 없어 tokenizer→training→evaluation을 끝까지 연결할 수 없다. 그래서 각 downstream task마다 **평가 가능한 Target 언어를 최소 3개씩** 확보하도록 7개를 선택했고, 지역·계통을 분산했다. 일반성보다 **traceability**를 우선한 결과다.

| --- | --- | --- |

### 3.3 학습 corpus와 sampling

| 항목 | 값 |
| --- | ---: |
| source raw sentences | 1,025,895,043 |
| MLM train total | 4,482,259 |
| — seen/head | 4,147,176 |
| — target7 | 335,083 |
| sampling factor α | 0.3 |

Sampling은 tokenizer 학습뿐 아니라 **실제 MLM 학습 corpus 생성에도 적용**된다. 각 language-script의 샘플링 확률은 $P_i \propto n_i^{\alpha}$($\alpha=0.3$)로, head 100만 대 tail 1천이면 $\alpha=1$의 약 1000:1이 $\alpha=0.3$에서 약 8:1로 완화된다. 이는 high-resource 독점을 막으면서 target overfitting도 피하는 절충이다.

### 3.4 평가 coverage 주의

- coverage 없는 group은 `0`이 아니라 `NA`.
- **Bible retrieval**은 50K에서도 모든 초기화가 Acc10 ~0.008로 floor(§6.4). 초기화 우열 근거로 쓰지 않는다.
- **Text classification(Taxi1500)**은 Glot500 정식 task이나 target test(PBC)가 로컬에 없어 **English(head)만** 평가된다(방법론이 아니라 데이터 한계).
- **NER**은 언어당 100문장 subset이라 diagnostic으로 해석한다.

## 4. 방법 (Method)

### 4.1 Tokenizer extension

XLM-R의 `sentencepiece.bpe.model`을 복사하고, v5.2 mixed corpus에서 unigram auxiliary model을 학습해 source에 없는 piece만 **SPM protobuf에 직접 append**한다(HuggingFace `add_tokens()`가 아니라 — Glot500 구현과 일치, unigram score 보존). 결과: vocab 250,002 → **366,666**(신규 116,664). fertility는 §6.1.

### 4.2 Embedding initialization

**공통 규칙.** source에 있는 token은 source row를 복사; 새 row만 method별 초기화; `<mask>`는 string identity로 remap; input embedding과 LM head weight tying 확인. **감사 수치**: source 250,002 · target 366,666 · new 116,664 · `<mask>` copy diff 0.0 · LM head tied.

**표 3. 다섯 초기화 방법.**

| Method | 정의 | 직관 |
| --- | --- | --- |
| `random` | 새 row를 $\mathcal{N}(0,0.02)$에서 sample | 무정보 baseline |
| `mean` | 기존 lexical embedding의 global mean 복사 | embedding 중심에서 안정적 시작(표면형 미사용) |
| `FVT` | 새 token을 source tokenizer로 재분해 → subtoken embedding 평균 | source 형태/철자 재사용 (main hypothesis) |
| `weighted FVT` | FVT 평균에 표면 길이 가중 | 긴 subtoken이 더 구체적 신호라는 가정 |
| `family-aware mean` | provenance의 language family source-token 평균 | 표면형 대신 계통 prior |

> `align` initializer는 이번 tokenizer에서 FVT와 byte-identical하게 collapse되어 5-way ablation에서 제외한다.

### 4.3 Continued MLM pretraining

| Item | Value |
| --- | --- |
| base model | `xlm-roberta-base` |
| objective | MLM (mask prob 0.15) |
| learning rate | 5e-5 |
| Adam | betas (0.9, 0.999), ε 1e-8 |
| schedule / weight decay | linear, warmup 0 / 0.0 (Trainer default) |
| max seq length | 512 |
| language sampling | α = 0.3 |
| checkpoint interval | 1,000 steps |
| convergence budget | 50,000 steps (5-way) |

다섯 방법 모두 동일 exposure로 50K-step까지 학습(prior + continuation)하고, 모든 최종 수치는 50K checkpoint 기준으로 보고한다.

## 5. 실험 프로토콜 (Experiment Protocol)

### 5.1 통제 변인

다섯 방법은 **오직 새 vocabulary row 초기화만** 다르고 tokenizer·corpus·objective·budget·evaluation runner는 모두 같다. 따라서 차이는 initialization과 training dynamics의 상호작용으로 해석한다.

### 5.2 평가 지표

| 평가 항목 | 무엇을 평가하나 | 해석 |
| --- | --- | --- |
| Pseudoperplexity | 해당 언어 문장을 얼마나 자연스럽게 예측하는지 | 낮을수록 좋음 |
| Sentence Retrieval Tatoeba | 영어 문장과 같은 의미의 다른 언어 문장을 찾는 능력 | Top-10 Acc. ↑ |
| Sentence Retrieval Bible | Bible 병렬 corpus에서 같은 의미 문장 찾기 | Top-10 Acc. ↑ |
| Text Classification | 영어 fine-tune 후 zero-shot 전이 | F1 ↑ |
| NER | 영어 NER 학습 후 다른 언어 개체명 인식 | F1 ↑ |
| Roundtrip Alignment | 여러 언어를 거쳐 원래 단어로 돌아오는지 | Accuracy ↑ |

### 5.3 Downstream task 정의 (입력→처리→출력→해석)

retrieval·similarity는 공통 문장 임베딩을 쓴다: 문장을 인코딩·forward → **layer 7**(0-index 8번째) 선택 → special/padding 제외 mean-pooling → L2 정규화(similarity는 centered normalize 추가).

- **Pseudoperplexity** (↓): target7 문장을 위치별로 `<mask>`하고 정답 token NLL을 지수화. fine-tune 없음. 낮을수록 자연스러운 예측.
- **Tatoeba retrieval** (↑): English↔Target 병렬 문장을 임베딩해 각 English 문장의 cosine 최근접 10개를 faiss로 검색, 정답 짝이 Top-10에 들면 hit. fine-tune 없음.
- **Bible retrieval** (↑): 위와 동일, 데이터가 PBC verse-aligned. 후보 pool이 크고 균질해 어렵다(§6.4).
- **NER** (↑): 인코더에 token classification head를 얹어 English WikiANN으로 fine-tune(10 epoch, LR 2e-5, max len 256) 후 target zero-shot, seqeval entity-level F1.
- **Roundtrip alignment** (↑): simalign 임베딩 word alignment로 여러 언어 cycle을 따라가 출발 단어로 복귀한 비율. fine-tune 없음.
- **Text classification (Taxi1500)** (↑): sequence classification head를 English로 fine-tune(30 epoch, LR 2e-5, effective batch 16) 후 평가, macro-F1. target test(PBC) 부재로 English만.

### 5.4 Fine-tuning / 평가 하이퍼파라미터 (재현)

| Task | Fine-tune | Epochs | LR | Batch(eff.) | Max len | Metric |
| --- | --- | ---: | ---: | ---: | ---: | --- |
| NER | English | 10 | 2e-5 | 8×4=32 | 256 | entity F1 |
| Text | English | 30 | 2e-5 | 8×2=16 | 100 | macro-F1 |
| Tatoeba/Bible | frozen | – | – | 128 | 512 | Acc10 (layer7, cosine) |
| Roundtrip | frozen | – | – | 25 | – | accuracy (simalign) |
| PPPL | frozen | – | – | – | – | weighted PPPL |

이 설정은 Glot500 recipe(NER LR 2e-5 Adam·English zero-shot, Text batch 16)와 일치한다. retrieval·roundtrip·PPPL은 학습이 없어 **초기화·MLM 차이만이 점수 차이의 원인**이다(fine-tune seed 분산 없음).

## 6. 결과 (Results)

결과는 세 층위로 제시한다: (§6.1) 전체 경향성, (§6.2) main table, (§6.3) step별 궤적. 모든 수치는 50K 기준.

### 6.1 전체 경향성과 best result

같은 tokenizer·corpus에서 초기화만 바꿔도 결과가 갈린다. **FVT 계열이 `random`·`mean`을 사실상 모든 지표에서 앞서고, 추가한 refinement(`weighted_fvt`·`family_mean`)가 수렴(50K) 시점에 plain `fvt`를 추월**한다. `mean`은 loss·PPPL에서 `random`보다도 나쁘다.

**표 4. Best-result 요약.**

| Metric (group) | dir | best init | best | random | Glot500-m |
| --- | :---: | --- | ---: | ---: | ---: |
| MLM loss (all) | ↓ | **weighted_fvt** | 2.73 | 3.12 | – |
| PPPL (tail) | ↓ | **family_mean** | 12.3 | 20.2 | 7.7 |
| Tatoeba (tail) | ↑ | **family_mean** | 36.1 | 33.4 | 45.7 |
| NER (tail) | ↑ | **family_mean** | 55.8 | 48.2 | 52.6 |
| Roundtrip (tail) | ↑ | **weighted_fvt** | 3.4 | 2.7 | 5.4 |
| Text (head/EN) | ↑ | fvt | 77.4 | 72.6 | 74.3 |
| Bible (tail) | ↑ | (floor) | ~0.9 | 0.8 | 14.7 |

특히 **`family_mean`은 PPPL·Tatoeba·NER 세 target 지표 모두 최고**이고, **NER(55.8)에서는 full Glot500-m(52.6)·XLM-R-L(53.9)까지 능가**한다. 50K 축소 모델이 계통 prior만으로 target 개체명 인식에서 baseline을 넘는다는 뜻이다.

### 6.3 Step별 궤적 (경향성)

**MLM loss 수렴.** 50K 최종: weighted_fvt 2.73 < fvt 2.76 < family_mean 2.91 < random 3.12 < mean 3.27. FVT 계열 최저, mean 최악.

**표 5. 10K→50K downstream 궤적(tail).**

| Method | PPPL 10K→50K | Tatoeba 10K→50K | Roundtrip 10K→50K |
| --- | ---: | ---: | ---: |
| fvt | 21.2 → 16.3 | 33.9 → 34.4 | 3.02 → 3.22 |
| weighted_fvt | 26.9 → 13.9 | 30.4 → 35.8 | 2.85 → 3.38 |
| family_mean | 34.2 → 12.3 | 30.7 → 36.1 | 2.61 → 3.16 |
| random | 28.4 → 20.2 | 33.0 → 33.4 | 2.61 → 2.67 |
| mean | 32.5 → 23.0 | 31.7 → 33.2 | 2.68 → 2.72 |

plain `fvt`는 10K에 이미 최고로 출발하나 **조기 포화**한다. `weighted_fvt`·`family_mean`은 10K에 더 나쁘게 출발하지만 **계속 개선되어 30~50K에서 fvt를 추월**한다. `random`·`mean`은 전 구간 하위. 즉 **정교한 초기화 prior의 이득은 학습 초반이 아니라 수렴 구간에 드러난다.**

### 6.4 Bible retrieval이 floor인 이유

Bible이 다섯 방법 모두 ~0.8로 붙는 것은 초기화 문제가 아니라 task 난이도 때문이다: (1) English verse 하나당 언어당 ~7–8천 절 중 정답을 찾아야 해(Tatoeba의 10배) 우연 상한이 매우 낮고, (2) 절이 짧고 균질(같은 성경 구조)해 문장 벡터가 서로 매우 비슷하며, (3) 50K budget으로는 verse 수준 정밀 정렬에 도달하지 못한다. full-scale Glot500-m(14.7)만 유의한 점수를 낸다. 따라서 초기화 비교에서는 제외한다.

### 6.5 Tokenizer fertility

확장 tokenizer는 Target7 tokens/word를 2.204 → 1.592로 **−27.75%** 낮춘다(언어별 12.4~34.9%). 다만 모든 초기화가 같은 tokenizer를 쓰므로 method 차이는 fertility로 설명되지 않는다.

## 7. 표현 공간 분석 (Analysis)

### 7.1 계통/문자 유사도

Target7이 모두 Latin script이므로 "같은 문자라서 가까운가"와 "계통이 가까워서 가까운가"를 분리해 본다. centered cosine을 쓴다(raw cosine은 대부분 0.9 근처라 anisotropy 영향이 큼).

**표 6. relation bucket별 centered cosine (FVT, 50K).**

| Relation | Pairs | centered cosine |
| --- | ---: | ---: |
| tail 같은 언어 | 350 | 0.455 |
| tail-tail 같은 family | 50 | 0.266 |
| tail-tail 같은 macro-family | 100 | 0.146 |
| tail-tail 다른 family | 900 | 0.074 |
| tail-head 같은 family | 1,050 | 0.052 |
| tail-head 다른 family·같은 Latn | 6,100 | −0.041 |
| tail-head 다른 family·다른 script | 1,856 | −0.063 |

유사도가 `같은 언어 > 같은 family > macro > 다른 family` 순으로 단조 감소한다. 같은 Latin script만 공유하는 경우(−0.041)보다 같은 family를 공유하는 경우(0.052)가 더 가깝다 — **script보다 family/typology 효과가 더 강한 설명력**을 가진다.

### 7.2 Step별 궤적: 구조는 조기 형성되고 평탄

10K→50K 구간에서 language/family geometry는 이미 형성되어 거의 변하지 않는다(예: 같은 언어 0.458→0.455, 같은 family 0.274→0.266로 평탄). cross-lingual semantic alignment(같은 의미 src↔eng)만 소폭 개선 후 포화한다(Tatoeba aligned 0.194→0.199, 20K에 이미 포화). 끝까지 단조 개선되는 것은 loss와 PPPL뿐이다. 따라서 "step이 갈수록 표현이 점점 더 분리된다"는 강한 주장은 이 구간에서는 성립하지 않으며, 단계적 창발이 있었다면 10K 이전 early phase였을 것이다(해당 snapshot 없음).

### 7.3 2D embedding map 해석

38개 언어 subset의 문장 임베딩 centroid를 2D PCA로 시각화한다(색=family, ▲=tail, ●=head). 같은 family 언어가 대체로 인접하고(Slavic·Romance·Austronesian 국소 군집), `dtp`·`bam`·`xav`는 주변 head가 적어 비교적 뚜렷이 분리된다. 반면 `csb`·`fur`·`ile`·`lij`는 유럽권 Latin 언어라 서로·인접 head와 겹친다. 각 tail centroid가 자기 계통 head 군집 방향에 자리하는 것은 vocabulary extension + continued pretraining이 새 언어를 관련 head의 표현 이웃으로 끌어온다는 관찰(related-language help)과 부합한다. 단 sentence-level은 완전한 분리가 아니고 2D는 거리 왜곡이 있어 보조적 근거로만 본다.

### 7.4 종합: 어떤 초기화가 가장 좋은가

| 관점 | 지표 | 최고 | 2위 |
| --- | --- | --- | --- |
| MLM 학습 | 50K training loss | **weighted_fvt** (2.73) | fvt (2.76) |
| MLM intrinsic | 50K target PPPL | **family_mean** (12.3) | weighted_fvt (13.9) |
| Downstream retrieval | Tatoeba | **family_mean** (36.1) | weighted_fvt (35.8) |
| Downstream labeling | NER | **family_mean** (55.8) | fvt (51.3) |
| Downstream alignment | Roundtrip | **weighted_fvt** (3.4) | fvt (3.2) |

`random`·`mean`은 모든 축에서 하위이고, 개선은 전부 FVT 계열에서 나온다. **`family_mean`이 target downstream의 최강자**(PPPL·Tatoeba·NER 3관왕, NER은 Glot500-m 능가)이나 전체 corpus loss는 3위다(target 특화, head 일반화엔 덜 유리). **`weighted_fvt`가 가장 균형 잡힌 선택**(loss·Roundtrip 최고, 나머지 상위권)이다. plain `fvt`는 "빠른 출발, 조기 포화" 프로파일이다. 결론적으로 target 성능이 목표면 `family_mean`, 전반적 균형이 목표면 `weighted_fvt`가 최선이다.

## 8. 논의·한계·결론 (Discussion, Limitations, Conclusion)

### 8.1 논의

이 프로젝트의 강점은 **통제의 엄격함**이다. tokenizer·corpus·objective·evaluation을 고정하고 초기화만 바꿨으므로 차이를 initialization과 training dynamics로 귀속할 수 있다. (1) tokenizer 확장은 fragmentation을 줄이지만 같은 tokenizer variant 사이에서도 차이가 유지되므로 novelty는 **초기화**에 있다. (2) FVT 계열이 이기는 이유는 새 token을 source subtoken 조합으로 설명해 형태·철자 정보를 재사용하기 때문이며, 50K loss와 target PPPL·Tatoeba·NER이 같은 방향을 가리켜 이 이득이 수렴까지 지속된다. (3) refinement가 plain FVT를 추월하는 이유는 균등 평균의 표현력이 제한되어 조기 포화하는 반면 길이 가중·계통 prior가 수렴 구간에서 계속 개선되기 때문이다 — 짧은 진단만으로 초기화를 평가하면 잘못된 결론에 이를 수 있다는 함의를 준다. (4) `mean`이 최악인 것은 모든 새 token을 embedding 중심 한 점에 몰아넣어 초기 분리를 방해하기 때문이다. (5) `family_mean`은 target에 특화되나 head 일반화엔 덜 유리하다.

### 8.2 한계

- Target7은 모두 Latin script라 script diversity 결론을 내릴 수 없다.
- task마다 target coverage가 다르다(NER 100문장 subset, Bible floor, Text head/EN-only).
- 계산 비용이 큰 일부 sequence-labeling 평가는 5개 초기화 전부에 완주하지 못해 제외했다(방법론이 아니라 compute/time 제약, future work).
- 50K에서 loss는 평탄화됐으나 완전 수렴으로 단정하지 않으며 절대 성능은 full-scale Glot500에 못 미친다.
- `weighted_fvt`·`family_mean`은 local variant이고, 단일 run이라 seed 분산은 충분히 평가되지 않았다.

### 8.3 결론

`xlm-roberta-base`에 Glot500-style vocabulary extension을 적용하고 새 embedding row 초기화를 다섯 방법으로 50K-step까지 통제 비교했다. (1) 확장 tokenizer는 Target7 fragmentation을 27.75% 줄였다. (2) 같은 tokenizer·corpus에서도 초기화에 따라 50K 수렴 loss가 갈렸고 FVT 계열이 최저, mean이 random보다도 높았다. (3) 50K downstream에서 refinement(`weighted_fvt`·`family_mean`)가 target PPPL·Tatoeba·NER·Roundtrip에서 최고였고 — `family_mean`은 NER에서 full Glot500-m까지 능가했다 — 학습 초반 앞서던 plain FVT를 수렴 구간에서 추월했다. (4) 표현 공간에서는 script보다 family 구조가 강하게 나타났고 이 구조는 조기에 형성돼 이후 평탄했다.

결론은 controlled setup으로 제한한다: **low-resource vocabulary adaptation에서 embedding initialization은 흔히 임의로 정해지지만 통제된 조건에서도 50K 수렴과 평가 지표를 실제로 바꾼다. source subtoken/계통 정보를 재사용하는 FVT 계열이 random·mean보다 지속적으로 유리하며, target 성능이 목표면 `family_mean`, 균형이 목표면 `weighted_fvt`가 최선이다.** 향후 과제는 더 넓은 script/language coverage, 다중 seed, target downstream coverage 확대다.
